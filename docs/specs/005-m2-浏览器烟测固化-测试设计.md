# 测试设计

## 背景
- 本文对应 [005-m2-浏览器烟测固化.md](/e:/AutoTest/docs/specs/005-m2-%E6%B5%8F%E8%A7%88%E5%99%A8%E7%83%9F%E6%B5%8B%E5%9B%BA%E5%8C%96.md)。
- 本轮目标是把上一轮临时联调脚本沉淀成仓库内测试，而不是继续做手工验证。

## 当前生效规则
- 继续遵守 `Spec -> TDD -> Execution`。
- 所有新增文档、注释统一使用中文。
- 这条浏览器烟测默认跳过，必须显式开启。

## 本轮测试目标
- 证明本地数据库迁移可以在烟测前自动收敛到最新状态。
- 证明本地前后端服务可以按需拉起，且不会破坏已运行服务。
- 证明真实登录态下，`M2` 第二刀主链可通过浏览器完整走通。
- 证明主链关键接口的真实返回结构仍符合页面消费预期。
- 证明烟测失败时可以自动保留定位证据，而不是只抛一个超时异常。

## 测试范围
- 仅覆盖一条主链：
  - 登录态建立
  - 用例生成页生成草稿
  - 标记审核并入库
  - 需求资产页查询和查看详情
  - 测试用例资产页查询和更新审核状态

## 非目标
- 不做视觉回归。
- 不做断言截图比对。
- 不做多浏览器兼容测试。
- 不覆盖历史模块。

## 用例设计

### 用例 1：默认执行 pytest 时烟测跳过
- 前置：
  - 未设置烟测环境变量
- 预期：
  - 测试文件被收集
  - 用例被跳过而不是强制执行

### 用例 2：显式开启后可自动完成迁移
- 输入：
  - 设置烟测环境变量
- 预期：
  - 烟测启动前执行 `aerich upgrade`
  - 数据库结构满足当前代码版本

### 用例 3：生成草稿成功
- 操作：
  - 打开用例生成页
  - 输入标题和内容
  - 点击生成
- 预期：
  - `generate_testcases` 返回成功
  - 页面出现草稿行

### 用例 4：审核并入库成功
- 操作：
  - 选中一条草稿
  - 标记已审核
  - 触发 `Ctrl + S`
- 预期：
  - `save_reviewed_cases` 返回成功
  - 返回 `testcase_count >= 1`

### 用例 5：需求资产页可查到刚入库的数据
- 操作：
  - 打开需求资产页
  - 按唯一标题搜索
  - 查看详情
- 预期：
  - 列表命中新数据
  - 详情接口返回关联测试用例
  - `module` 值正确返回

### 用例 6：测试用例资产页可更新审核状态
- 操作：
  - 打开测试用例资产页
  - 按唯一标题搜索
  - 修改审核状态并保存
- 预期：
  - `update_review_status` 返回成功
  - 返回的新状态与提交值一致

### 用例 7：失败时自动保留页面证据
- 操作：
  - 让烟测在任一页面步骤故意失败
- 预期：
  - 失败目录被保留
  - 至少包含：
    - 截图
    - 页面 HTML
    - 控制台日志
    - 页面异常
    - 请求失败记录
    - 失败步骤说明

## 失败暴露要求
- 服务不可达时，报错中必须能看出是哪一端失败。
- 登录失败时，报错中必须能看出是接口失败而不是页面定位失败。
- 页面元素未出现时，报错中必须能看出处于哪一页、哪一步。
- 接口返回非 200 时，必须直接抛出原始返回体，避免吞错。
- 页面失败证据必须可以在本地目录中直接回看。

## 本轮测试目标
- 证明自动化草稿保存结果会返回 `target_route_query`。
- 证明测试用例资产页可以展示自动化摘要，而不是只有生成按钮。
- 证明测试用例资产页可以直接打开已有草稿，不误生成新草稿。
- 证明自动化草稿页可以直接打开历史草稿。
- 证明 API / Web / App 目标页可以根据 query 自动定位到新生成资产。
- 证明目标资产不存在时，页面提示明确，不静默失败。

## 用例设计

### 用例 1：保存结果包含精确跳转参数
- 前置：
  - 已存在一条待保存自动化草稿
- 操作：
  - 调用保存接口
- 预期：
  - 返回 `target_route`
  - 返回 `target_route_query`
  - `target_route_query` 至少包含：
    - API：`asset_id`
    - Web：`menu_id`
    - App：`menu_id`
    - 通用：`draft_id`

### 用例 2：测试用例分页结果包含自动化摘要
- 操作：
  - 查询测试用例分页列表
- 预期：
  - 每条用例存在 `automation_summary`
  - `automation_summary.latest_draft_count` 正确
  - `automation_summary.latest_items` 能反映最新草稿状态与目标资产编号

### 用例 3：测试用例页从已有草稿继续编辑
- 前置：
  - 某条测试用例已存在 API / Web / App 草稿
- 操作：
  - 点击“继续编辑”
- 预期：
  - 跳转到自动化草稿页
  - query 中携带已有 `draft_id`
  - 自动化草稿页打开的是旧草稿，而不是重新生成

### 用例 4：自动化草稿页打开历史草稿
- 前置：
  - 某条测试用例已有多条自动化草稿
- 操作：
  - 点击历史列表中的“打开”
- 预期：
  - 当前页切换到对应 `draft_id`
  - `draftJson`、状态、告警同步刷新

### 用例 5：自动化草稿页查看目标资产
- 前置：
  - 某条历史草稿已保存到目标资产
- 操作：
  - 点击“查看目标资产”
- 预期：
  - 跳转到对应目标页
  - 携带 `target_route_query`

### 用例 6：API 目标页按 query 定位资产
- 前置：
  - 路由参数包含 `asset_id`
- 预期：
  - 页面加载后自动查询
  - 对应 `Api_script` 记录被高亮、选中或自动进入编辑态

### 用例 7：Web 目标页按 query 定位资产
- 前置：
  - 路由参数包含 `menu_id`
- 预期：
  - 对应菜单节点自动选中
  - 对应脚本 tab 自动打开

### 用例 8：App 目标页按 query 定位资产
- 前置：
  - 路由参数包含 `menu_id`
- 预期：
  - 对应菜单节点自动选中
  - 对应脚本页签自动打开

### 用例 9：目标资产不存在时提示明确
- 前置：
  - query 指向一个已删除或不存在的资产
- 预期：
  - 页面提示“目标资产不存在”或同等级明确文案
  - 页面保持可继续操作，不进入空白异常状态

## 失败场景设计

### 失败场景 1：已有草稿继续编辑时又生成了新草稿
- 预期：
  - 测试直接失败
  - 后端草稿数量不能增长

### 失败场景 2：保存成功但没有 `target_route_query`
- 预期：
  - 第二刀不算通过

### 失败场景 3：测试用例资产页摘要与真实草稿记录不一致
- 预期：
  - 列表页测试失败

### 失败场景 4：目标页拿到 query 但没有定位到目标资产
- 预期：
  - 页面级验证失败

### 失败场景 5：目标资产不存在时没有提示
- 预期：
  - 页面级验证失败

### 失败场景 6：自动化草稿页打开历史草稿时，页面仍保留旧内容
- 预期：
  - 历史打开验证失败

  - `target_route_query`
  - 自动化摘要
  - 已有草稿重开

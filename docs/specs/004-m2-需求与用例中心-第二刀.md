# 规格说明

## 背景
- `003-m2-需求与用例持久化-第一刀.md` 已经解决了“已审核草稿可以保存为平台资产”。
- 当前缺口不在“能不能保存”，而在“保存后如何被管理”：
  - 需求资产和测试用例资产仍然主要挂在“用例生成”页里查看。
  - 缺少独立的需求资产列表页和测试用例列表页。
  - 缺少分页、筛选、详情查看、审核状态再维护等最小管理能力。
- 当前测试用例编辑交互仍然偏表单化或基础表格化，不适合高频录入、批量修订和纯键盘操作。
- 如果要把测试用例真正做成平台资产，而不是一次性草稿，测试用例编辑区必须升级成高性能数据网格，而不是继续沿用基础 `el-table`。
- 如果现在直接进入 `M3. 自动化草稿生成`，会导致上游资产层还没站稳就开始往下游扩展，后面会越来越难治理。

## 目标
- 把需求资产和测试用例资产从“生成页附属能力”提升成平台内可独立进入、可查看、可筛选、可维护的一等实体。
- 在不引入复杂审批流的前提下，补齐最小“资产管理闭环”。
- 为后续 `M3. 自动化草稿生成` 提供稳定的输入资产入口。
- 把测试用例编辑体验升级成面向高频录入的网格编辑模式，支持键盘优先操作和快捷入库。

## 非目标
- 本次不进入自动化草稿生成。
- 本次不做文件上传、文档解析、RAG 或知识库。
- 本次不做复杂审批流、多级审核流、版本对比。
- 本次不做批量编辑、批量删除、回收站。
- 本次不把整个前端仓库的所有 `el-table` 全部替换掉。
- 本次只约束“用例生成”“测试用例资产管理”这两个测试用例相关编辑场景，不扩展到系统内其他历史列表页。

### 受影响页面
- 新增一级业务菜单下的“需求资产”页面。
- 新增一级业务菜单下的“测试用例”页面。
- 保留现有“用例生成”页面，继续作为生成入口。
- “用例生成”页面中的测试用例草稿编辑区，升级为高性能数据网格。

### 受影响接口/模型
- 继续复用现有：
  - `Requirement`

## 输入输出

### 输入
- 需求资产列表筛选条件：
  - 关键词
  - 来源类型
  - 状态
  - 分页参数
- 测试用例列表筛选条件：
  - 关键词
  - 所属需求
  - 审核状态
  - 目标类型
  - 分页参数
- 测试用例审核状态更新：
  - `testcase_id`
  - 新审核状态
- 网格内测试用例编辑字段：
  - `module`
  - `title`
  - `preconditions`
  - `steps`
  - `expected_results`
  - `priority`

### 输出
- 需求资产分页结果：
  - 基本信息
  - testcase 数量
  - 最近一次生成模式
  - 创建时间
- 测试用例分页结果：
  - 基本信息
  - 所属需求
  - 审核状态
  - 目标类型
  - 自动化适配标记
- 详情结果：
  - 需求详情
  - 关联测试用例明细
- 网格编辑输出：
  - 只展示 6 个业务编辑列
  - 追溯、审核、主键等字段作为隐藏元数据保留

### 状态变化
- 测试用例允许在资产页做最小审核状态维护：
  - `草稿`
  - `已审核`
  - `已拒绝`
- 本次不引入更复杂的审批节点。

## 字段与入库基线

### 主编辑网格可见字段
- `所属模块 (module)`：字符串
- `用例标题 (title)`：字符串
- `前置条件 (preconditions)`：前端单个多行文本，后端按换行拆分为数组入库
- `测试步骤 (steps)`：前端单个多行文本，后端按换行拆分为数组入库
- `预期结果 (expected_results)`：前端单个多行文本，后端按换行拆分为数组入库
- `优先级 (priority)`：下拉选，枚举值 `P0 / P1 / P2 / P3`

### 隐藏元数据
- 以下字段继续存在，但不进入主编辑网格可见列：
  - `id`
  - `requirement_id`
  - `generation_job_id`
  - `category`
  - `review_status`
  - `user_id`
  - `create_time`
  - `update_time`

### 次级信息
- `category` 保留为“用例类型”，不从数据模型中移除。
- 在本切片中，`category` 不进入主编辑网格，但可在详情区、抽屉、侧栏或次级信息区展示。

### 与现有模型的兼容约束
- 当前后端 `Testcase` 模型中存在历史字段：
  - `category`
  - `preconditions`
  - `expected_results`
- 本切片开始：
  - `module` 新增为真正的业务字段
  - `category` 保留为用例类型
  - `preconditions`、`steps`、`expected_results` 继续沿用现有字段命名
- 第二刀实现时，重点不是重命名 `preconditions / expected_results`，而是补齐 `module` 字段，并在前端网格层只暴露主编辑字段。
- 前端主编辑网格只认这 6 个业务字段：
  - `module`
  - `title`
  - `preconditions`
  - `steps`
  - `expected_results`
  - `priority`

## 方案摘要
- 后端在现有 `requirement` / `testcase` 模块上继续扩展列表查询和状态更新接口，不新开并行模块。
- 前端新增两个独立业务页面：
  - `需求资产`
  - `测试用例`
- 需求资产页负责“找需求、看关联用例”。
- 测试用例页负责“找用例、看审核状态、做最小状态维护”。
- 现有“用例生成”页继续负责“输入需求 -> 生成草稿 -> 人工审核 -> 保存资产”，但其中的草稿编辑区升级为 `vxe-table` 数据网格。
- 本切片内测试用例相关编辑网格强制选用 `vxe-table`，不选 `ag-grid`，也不继续用基础 `el-table` 承担编辑职责。
- 网格中的“前置条件 / 测试步骤 / 预期结果”使用单个多行文本编辑，但提交给后端时按换行拆分为数组。

## 交互约束

### 高性能数据网格
- 测试用例编辑与资产管理场景中的主编辑表格，强制使用 `vxe-table`。
- 本条约束只作用于本切片新增或改造的测试用例编辑场景，不追溯要求全仓历史 `el-table` 统一替换。

### 全键盘导航能力
- 必须开启 `vxe-table` 的 `keyboard-config`。
- 必须支持：
  - 上下左右方向键切换单元格焦点
  - `Enter` 或 `F2` 进入编辑模式
  - `Esc` 退出编辑模式

### 底部操作栏
- 页面右下角常驻 `一键入库` 按钮。
- 按钮文案需明确标注快捷键：`Ctrl + S`。
- 页面必须拦截浏览器默认保存行为，并将 `Ctrl + S` 绑定到当前测试用例网格的入库动作。

## 关键约束
- 继续遵守 `Spec -> TDD -> Execution`，本文件只用于需求对齐，不作为实现开始信号。
- AI 能力仍然只能通过统一 AI Gateway 进入，资产管理页不得直连模型。
- 不为资产管理再造一套重复数据模型，必须复用现有 requirement/testcase/generation job。
- 新增配置不得散落在页面或业务模块。
- 所有新增文档和代码注释必须使用中文。
- 不允许把“高性能数据网格”的要求扩写成“全仓所有表格全部迁移”。

## 风险点
- 风险 1：如果列表和详情字段定义不稳，前后端会反复对齐，造成接口来回改。
- 风险 2：如果审核状态允许在多个页面修改，容易出现生成页、资产页显示不一致。
- 风险 3：如果现在就加入批量操作、版本对比，会让本切片失控，拖慢进入 `M3` 的时间。
- 风险 4：如果不先收敛 `module` 与现有 `category` 的职责边界，后续会出现“模块”和“用例类型”混用。
- 风险 5：引入 `vxe-table` 后，如果键盘行为和现有快捷键体系冲突，页面交互会变得不稳定。

## 验收标准
- [ ] 用户可以从一级业务菜单进入“需求资产”页面并看到分页列表
- [ ] 用户可以按关键词、来源类型、状态筛选需求资产
- [ ] 用户可以查看某个需求下关联的测试用例明细
- [ ] 用户可以从一级业务菜单进入“测试用例”页面并看到分页列表
- [ ] 用户可以按关键词、所属需求、审核状态、目标类型筛选测试用例
- [ ] 用户可以在测试用例页更新单条测试用例的审核状态
- [ ] 测试用例主编辑网格使用 `vxe-table`，不再由基础 `el-table` 承担编辑职责
- [ ] 主编辑网格可见列包含且仅包含：`module`、`title`、`preconditions`、`steps`、`expected_results`、`priority`
- [ ] 网格支持方向键切换焦点、`Enter/F2` 进入编辑、`Esc` 退出编辑
- [ ] 页面右下角常驻 `一键入库` 按钮，且 `Ctrl + S` 可触发入库
- [ ] 现有“用例生成 -> 保存资产”链路不回归
- [ ] 至少有后端 pytest 和前端类型校验覆盖本阶段改动

## 失败场景
- [ ] 分页参数缺失或非法时，接口仍能安全返回默认结果
- [ ] 查询条件为空时，列表仍能正常返回
- [ ] 需求已存在但关联 testcase 为空时，详情页不报错
- [ ] 测试用例审核状态更新到非法值时，后端拒绝保存
- [ ] 资产页修改审核状态后，生成页再次查看时状态不一致
- [ ] 新页面路由加入后，现有菜单和动态路由不回归
- [ ] 网格录入 `priority` 非法值时，页面或后端必须拒绝保存
- [ ] `Ctrl + S` 未被拦截，触发浏览器默认保存行为
- [ ] 多行文本单元格编辑退出后，内容丢失或换行被错误截断
- [ ] `module`、`category` 职责边界未收敛，导致保存后展示信息错位

  - 新增的列表查询和状态维护接口
- 已完成的 `M1` 和 `M2` 第一刀不得回退。

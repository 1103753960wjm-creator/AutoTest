# 规格说明

## 背景
- `004-m2-需求与用例中心-第二刀.md` 已经完成了核心实现，但上一轮浏览器联调仍依赖临时脚本。
- 临时脚本虽然能验证真实链路，但不可沉淀、不可复用，也不适合作为后续回归基线。
- 当前最需要的不是继续加功能，而是把已经跑通的主链固化成仓库内可重复执行的烟测。

## 目标
- 把 `用例生成 -> 已审核入库 -> 需求资产查看 -> 测试用例资产审核状态更新` 固化为仓库内浏览器烟测。
- 让这条烟测可以在本地按需重复执行，而不是继续依赖一次性命令。
- 在不影响日常开发速度的前提下，为后续 `M3` 改造保留真实回归入口。

## 非目标
- 本次不新增任何业务功能。
- 本次不补前端组件单测体系。
- 本次不接入 CI 自动执行。
- 本次不覆盖多浏览器、多账号、多环境矩阵。

## 范围
- 后端测试目录：
  - `l-tester-master/tests`
- 项目规格文档：
  - `docs/specs`

## 方案摘要
- 新增一条基于 `pytest + Playwright` 的浏览器烟测。
- 烟测默认跳过，只有显式设置环境变量后才执行。
- 烟测内部负责：
  - 确保数据库迁移已执行
  - 按需启动本地前后端服务
  - 调登录接口获取真实 token
  - 注入本地登录态
  - 走完整业务主链
- 烟测失败时自动保留：
  - 页面截图
  - 页面 HTML
  - 浏览器控制台日志
  - 页面异常
  - 请求失败记录
- 烟测结束后，只清理它自己启动的本地进程，不影响外部已运行服务。

## 运行前提
- 本机可访问本地 MySQL。
- 本机已安装前端依赖。
- 本机已安装可用于 Playwright 的浏览器内核，默认走 `msedge`。
- 本机允许启动浏览器和前端开发服务。
- 如需自定义失败产物目录，可设置 `LT_SMOKE_ARTIFACT_DIR`。

## 验收标准
- [ ] 仓库内存在独立的浏览器烟测文件
- [ ] 默认直接执行 `pytest` 时，这条烟测不会强制参与主测试集
- [ ] 显式开启烟测后，可以完成：
  - 生成草稿
  - 标记审核
  - `Ctrl + S` 入库
  - 需求资产查询和详情查看
  - 测试用例资产查询和审核状态更新
- [ ] 烟测失败时，能自动保留截图、页面 HTML、控制台日志和失败步骤

## 失败场景
- [ ] 前端服务未启动且无法自动拉起
- [ ] 后端服务未启动且无法自动拉起
- [ ] 数据库迁移未执行，导致入库时报表不存在
- [ ] 登录接口失败，无法建立真实登录态
- [ ] 生成草稿接口失败
- [ ] 入库接口失败
- [ ] 需求资产列表或详情接口失败
- [ ] 测试用例审核状态更新接口失败

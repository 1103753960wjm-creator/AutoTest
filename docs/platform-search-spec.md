# 平台全局搜索骨架规范

更新时间：2026-03-18

## 1. 目标

阶段 1.4 的目标不是建设完整搜索系统，而是在现有平台壳下建立第一版统一搜索入口与结果分组骨架。

本轮重点：

- 顶部全局搜索入口升级为真实可交互入口
- 第一版优先支持页面 / 菜单 / 入口搜索
- 谨慎接入 1 到 2 类低风险轻资产
- 与最近访问 / 收藏能力分离，但允许搜索结果中展示少量最近访问
- 为未来服务端搜索保留 store 与动态适配边界

## 2. 边界

本轮明确不做：

- 不重写 Layout
- 不新增第二套页面头部体系
- 不重做 Home 工作台结构
- 不把最近访问 / 收藏面板与搜索面板合并
- 不建设完整全文检索系统
- 不提前做复杂搜索筛选、收藏分组、搜索历史中心

## 3. 入口形态

第一版搜索入口固定挂在顶部全局栏。

交互形态：

- 顶栏搜索按钮打开
- `Ctrl / Cmd + K` 打开
- `Esc` 关闭

选型理由：

- `command palette` 风格轻量对话框更适合结果分组和键盘交互
- 比 popover 更适合键盘导航
- 比 drawer 更轻，不会演变成第二套个人效率中心

## 4. 搜索范围

### 4.1 静态索引

静态索引优先来自两个真源：

- `frontend/src/config/navigation.js`
- `frontend/src/router/index.js` + `frontend/src/router/route-meta.js`

第一版纳入：

- 一级模块入口
- 二级页面入口
- 常用页面和模块入口

静态页不纳入：

- 动态参数详情页
- 认证页
- 明确隐藏且无法稳定直达的页面

### 4.2 最近访问结果组

最近访问不是搜索面板的主体系统，但可以作为搜索中的辅助结果组。

规则：

- 搜索框为空时，展示最近访问
- 有关键字时，也保留少量匹配的最近访问结果
- 有关键字时最多展示 3 条，避免冲乱主结果

### 4.3 轻资产搜索

第一版只接入两类低风险对象：

- 测试设计项目
- 测试用例

原因：

- 已有稳定列表接口
- 已有明确详情页路由
- 点击搜索结果可以直接跳转

当前不接入：

- 接口自动化项目
- 仅能在弹框中查看详情的对象
- 需要新增搜索专用后端接口的复杂实体

## 5. 结果分组

第一版结果分组固定为：

1. 页面 / 菜单 / 入口
2. 最近访问
3. 轻资产

说明：

- 页面 / 菜单命中优先
- 轻资产命中其次
- 最近访问始终作为辅助结果组存在，但数量受控

## 6. 排序规则

静态结果不只按简单字符串包含排序。

第一版固定优先级：

1. 标题完全命中
2. 标题前缀命中
3. 标题包含命中
4. 模块名命中
5. 关键词或路由命中
6. 描述模糊命中

在同级命中里，再按基础优先级和标题排序收敛。

## 7. 动态搜索触发规则

轻资产搜索只在关键字达到最小长度时触发。

当前规则：

- 关键字 `< 2`：不触发动态轻资产请求
- 关键字 `>= 2`：触发项目 + 测试用例轻量搜索

目的：

- 避免空搜索打接口
- 避免单字搜索产生过多噪音
- 避免打开弹层后进入请求风暴

## 8. 键盘交互

第一版最小键盘交互：

- `Ctrl / Cmd + K`：打开搜索
- `Esc`：关闭搜索
- `ArrowUp / ArrowDown`：切换当前选中结果
- `Enter`：打开当前选中结果

## 9. 数据结构

统一搜索结果项结构：

```js
{
  id: 'route:/ai-generation/projects',
  group: 'pages',
  groupLabel: '页面 / 菜单 / 入口',
  type: 'page',
  title: '项目管理',
  description: '测试设计 / 列表页',
  route: '/ai-generation/projects',
  module: 'test-design',
  moduleLabel: '测试设计',
  pageType: 'list',
  pageTypeLabel: '列表页',
  icon: 'folder',
  source: 'static'
}
```

轻资产结果项沿用同一结构，只调整：

- `type`
- `source`
- `summary`

## 10. 与最近访问链路的关系

搜索结果打开页面后，仍然走统一路由链。

因此：

- 从搜索结果打开页面
- 仍会进入 `router.afterEach`
- 会自然写入最近访问

这保证了搜索、最近访问、Home 快捷继续三者仍然是同一条平台效率链路。

## 11. 后续扩展边界

后续如果接入服务端搜索，不直接改页面层，而是优先扩展：

- `frontend/src/stores/platform-search.js`

建议扩展方向：

- 服务端统一搜索接口适配
- 搜索历史
- 结果高亮
- 更多轻资产类别
- 权限感知搜索结果

# 测试设计对象层收口规范

## 1. 文档目标

本规范用于落实阶段 2 - 任务 2.1：测试设计对象层收口（v2 修正版）。

本轮目标不是重构 AI 生成流程，也不是建设自动化草稿中心，而是先把测试设计主链上的对象层、关系层和摘要语义收稳，让后续 2.2 与 2.3 有稳定挂接点。

## 2. 生效范围

本轮覆盖以下页面与接口语义：

- 测试设计项目列表
- 测试设计项目详情
- 测试用例列表
- 测试用例详情
- 测试用例编辑
- 需求分析页
- 生成结果列表
- 任务详情页
- 执行详情页（轻量来源位）

本轮重点落点：

- 对象层级固定
- 页面关系收清
- 摘要语义统一
- AI 来源位预留
- 自动化状态位预留

## 3. 对象层级

### 3.1 第一层：测试设计项目

对象定位：

- 测试设计源头对象
- 承接项目内需求输入、AI 生成活动、测试用例资产与后续自动化挂接

本轮对象字段语义：

- `name`：项目名称
- `description`：项目描述
- `status`：项目状态
- `updated_at`：最近更新时间
- `testcase_count`：项目下测试用例数量
- `requirement_summary`：需求分析摘要位
- `ai_generation_summary`：AI 生成摘要位
- `automation_summary`：自动化摘要位

### 3.2 第二层：需求分析对象

对象定位：

- 需求输入与分析过程对象
- 当前以“需求文档 / 需求文本输入 + 分析状态 + 配置来源摘要”共同承接

本轮对象字段语义：

- `input_content_summary`：输入文本或上传内容摘要
- `analysis_status`：分析状态
- `config_source_summary`：模型 / Prompt / 生成配置摘要
- `last_analysis_at`：最近分析时间
- `generation_entry`：生成任务入口

### 3.3 第三层：生成任务对象

对象定位：

- AI 生成过程对象
- 当前以 `TestCaseGenerationTask` 为主对象

本轮对象字段语义：

- `task_id`
- `title`
- `status`
- `project / project_name`
- `writer_model_name / reviewer_model_name`
- `writer_prompt_name / reviewer_prompt_name`
- `generation_config_summary`
- `created_at / completed_at`
- `result_count`
- `error_message`
- `save_status_summary`

说明：

- 当前任务模型未持久化任务级 `generation_config` 外键。
- 本轮允许用“当前活跃生成配置”作为推断摘要，并显式标注为推断来源。

### 3.4 第四层：生成结果对象

对象定位：

- AI 生成产物的中间对象
- 当前主链以“任务内生成结果批次 + 解析后的结果项”承接

本轮对象字段语义：

- `result_count`
- `result_summary`
- `is_confirmed`
- `is_saved_to_testcases`
- `source_task_id`
- `source_project`

说明：

- 由于当前主流程仍以 `final_test_cases` 文本结果为主，本轮不重做独立结果存储模型。
- 页面层将“任务结果批次”明确展示为生成结果对象入口，避免继续把它只当工具输出。

### 3.5 第五层：测试用例对象

对象定位：

- 测试设计资产对象
- 作为后续评审、自动化草稿、执行回链的稳定资产

本轮对象字段语义：

- `project`
- `title`
- `priority`
- `test_type`
- `steps / expected_result`
- `source_summary`
- `review_summary`
- `automation_summary`
- `updated_at`
- `generation_source_summary`

说明：

- 当前历史数据未稳定持久化“人工 / AI 来源”。
- 本轮允许通过新增 AI 导入时写入轻量来源标记实现“可识别即展示”，无法识别时统一显示“来源未记录”。

### 3.6 第六层：执行相关对象

对象定位：

- 设计资产的下游执行对象
- 本轮仅承接来源展示位，不重做执行链路

本轮展示语义：

- 关联项目
- 关联计划 / 执行轮次
- 来源设计资产预留位
- 自动化来源预留位

## 4. 页面关系

本轮页面关系固定如下：

1. 项目页是设计源头对象页
2. 需求分析页是分析对象页
3. 任务详情页是生成任务对象页
4. 生成结果列表页是生成结果批次页
5. 测试用例详情页是设计资产详情页
6. 执行详情页只承接来源回链预留位

主关系链固定为：

`测试设计项目 -> 需求分析对象 -> 生成任务对象 -> 生成结果对象 -> 测试用例对象 -> 执行对象`

## 5. 页面展示原则

### 5.1 项目详情页

必须体现：

- 项目基础摘要
- 测试设计资产数量
- 需求分析摘要
- AI 生成摘要
- 自动化预留摘要
- 下游入口：需求分析、生成结果、测试用例

### 5.2 测试用例详情页

必须体现：

- 资产摘要
- 所属项目
- 来源摘要
- 自动化状态位
- 评审状态位
- 设计内容主体

### 5.3 测试用例编辑页

必须体现：

- 正在编辑哪个资产
- 属于哪个项目
- 来源上下文与保存后回跳规则

### 5.4 需求分析页

必须体现：

- 当前分析对象的输入方式
- 当前关联项目
- 当前配置来源摘要
- 当前任务状态 / 最近结果入口
- 下游挂接说明

### 5.5 生成结果列表页

必须体现：

- 结果批次列表
- 来源任务
- 来源项目
- 结果数量
- 保存状态
- 进入任务详情入口

### 5.6 任务详情页

必须体现：

- 任务对象摘要
- 来源项目
- 模型 / Prompt / 生成配置摘要
- 结果数量
- 保存状态
- 生成结果列表主体

### 5.7 执行详情页

必须体现：

- 关联项目
- 设计来源预留位
- 自动化来源预留位

## 6. 关系与接口语义

### 6.1 项目与需求分析

- 项目通过需求文档 / 分析活动形成需求分析摘要
- 项目详情需要能看到是否已有需求输入、是否已有分析活动

### 6.2 项目与生成任务

- 生成任务应明确关联项目
- 项目详情需要能看到最新生成任务摘要
- 需求分析页发起生成时应优先带项目上下文

### 6.3 生成任务与生成结果

- 任务负责承接过程状态
- 结果负责承接产物摘要
- 页面层允许以“任务结果批次”形式展示结果对象，但必须清楚区分“这是结果批次，不是项目或用例资产”

### 6.4 生成结果与测试用例

- 保存为正式测试用例后，生成结果对象需要有“已保存”语义
- 测试用例对象需要有“生成来源摘要位”

### 6.5 测试用例与执行

- 本轮只保留来源回链预留位
- 不重做执行全链路闭环

## 7. AI 来源位语义

本轮统一使用以下语义：

- `ai_generation_summary`：项目级 AI 生成摘要
- `generation_config_summary`：任务级生成配置摘要
- `source_summary`：用例来源摘要
- `generation_source_summary`：用例生成来源摘要

展示规则：

- 可识别 AI 来源时，展示 `AI 生成 / 来源任务 / 来源项目`
- 无法识别时，展示 `来源未记录`
- 推断值必须在文案上体现为“当前活跃配置”或“推断摘要”，避免伪装成强真值

## 8. 自动化状态位语义

本轮统一使用以下语义：

- `automation_summary.status = pending`：已预留，未进入自动化草稿中心
- `automation_summary.status = draft_pending`：预留给 2.3 自动化草稿中心
- `automation_summary.label`：统一展示口径
- `automation_summary.detail`：说明该位仅为对象层挂接位，不代表已完成自动化落地

本轮所有自动化状态位都不驱动真实自动化流程。

## 9. 深链接与回跳

本轮继续复用阶段 1 的深链接与回跳规则：

- 对象身份放 `params`
- 来源上下文放 `query`
- 回跳顺序仍为：合法 `fromPath` -> 所属列表页 -> `router.back()`

本轮新增纳入对象页范围的详情页，应继续遵守同一套规则，不新增第二套返回机制。

## 10. 本轮暂不处理

以下内容明确不在 2.1 范围内：

- AI 模型配置流程重构
- Prompt 配置流程重构
- AI 助手页重构
- 自动化草稿中心
- API / Web / App 自动化资产正式落地
- 执行结果全链路闭环
- 评审 / 版本体系大改
- Layout 与第二套页面头部体系

## 11. 验收标准

- 项目、需求分析、生成任务、生成结果、测试用例对象边界更清晰
- 项目详情页具备设计源头对象页结构
- 测试用例详情页具备设计资产对象页结构
- 需求分析页、生成结果页、任务详情页具备生成链对象页结构
- 项目、任务、用例页面出现 AI 来源位和自动化状态位
- 后续 2.2 与 2.3 有明确挂接点
- 不破坏阶段 1 的平台壳、唯一页面头部、深链接与回跳基线

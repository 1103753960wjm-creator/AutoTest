# Home 工作台改版说明

## 1. 目标

本轮将 `Home` 从“模块入口集合页”升级为统一平台壳下的工作台首页。

当前目标：

- 让首页先表达“现在可以继续什么工作”
- 保留模块入口，但不再把首页做成单纯入口页
- 继续服从统一 Layout 和 `PlatformPageHeader`
- 复用现有共享组件，不再生长新的首页平行体系

## 2. 区块职责

### 2.1 页面头部

- 继续由 `PlatformPageHeader` 承接
- `Home` 通过 `usePlatformPageHeader` 注册描述、辅助信息和页面级动作
- 页面主体不再重复渲染平台级标题、面包屑和导航信息

### 2.2 欢迎区 / 当前上下文

- 承接用户欢迎语
- 承接当前工作说明
- 承接轻量项目上下文占位提示
- 只做工作台欢迎区，不做页面 Banner

### 2.3 平台概览

- 使用 `StatCard`
- 控制在 3 到 4 张
- 只做平台级轻概览，不复制各模块 dashboard

### 2.4 我的工作

- 使用 `QuickActionCard`
- 展示待处理、待跟进和继续当前工作
- 当前以轻量占位结构为主

### 2.5 核心模块入口

- 使用 `QuickActionCard`
- 数据来自平台导航真源
- 不再重新维护一套首页专属模块配置

### 2.6 快捷继续

- 使用 `RecentList`
- 本轮先提供稳定接入位
- 后续阶段 1.3 再接真实最近访问和最近处理对象

### 2.7 风险提醒 / 最近动态

- 使用 `RecentList`
- 当前展示轻量提醒和平台动态占位结构
- 不提前扩张成平台级运营看板或执行聚合中心

## 3. 真数据与占位边界

### 3.1 当前真实数据

- 当前用户：来自 `userStore.user`
- 核心模块入口：来自平台导航真源
- 已接入工作域数量：来自导航真源计算
- 页面头部信息：通过 `usePlatformPageHeader` 注入 Layout

### 3.2 当前占位数据

- 我的待处理
- 待跟进
- 快捷继续列表
- 风险提醒 / 最近动态列表
- 项目上下文展示

说明：

- 这些内容当前只提供稳定结构和自然接入口
- 不在本轮提前实现最近访问、收藏和全局搜索真实能力

## 4. 后续自然接入点

### 4.1 阶段 1.3 最近访问 / 收藏

- 可直接替换 `快捷继续` 区的占位数据源
- 无需改 Layout 和头部体系

### 4.2 阶段 1.4 全局搜索

- 保持平台全局头部入口不变
- `Home` 只承接搜索结果回流后的“继续工作”场景，不承担全局搜索壳层职责

## 5. 当前约束

- 不重写 Layout
- 不重写 `PlatformPageHeader`
- 不新增第二套头部体系
- 不提前抽象重型工作台框架

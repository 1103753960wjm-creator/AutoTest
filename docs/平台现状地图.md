# 平台现状地图

## 0. 判定口径

本地图用于给后续所有改造提供唯一真相，判定顺序如下：

1. 前端正式可达页面：以 `frontend/src/router/index.js` 为准。
2. 前端模块菜单：以 `frontend/src/layout/index.vue` 和 `frontend/src/views/Home.vue` 为准。
3. 后端 app 归属：以 `backend/urls.py` 和各 `apps/*/urls.py` 为准。
4. 目录里存在但未被主路由引用的页面：标记为“候选 / 遗留视图”，不算当前正式导航。

## 5. 重复和模糊入口

### 5.1 配置类页面被拆散在多处

- 正式入口叫“配置中心”，但配置页文件实际散落在：
  - `frontend/src/views/requirement-analysis/*`
  - `frontend/src/views/configuration/*`
  - `frontend/src/views/app-automation/settings/*`
  - `frontend/src/views/ui-automation/notification/*`
- 结论：`/configuration` 是“聚合菜单”，不是物理上的统一配置模块。

### 5.2 “谁是系统管理”目前并不清晰

- Vue 前端没有独立的系统管理 / 用户管理一级模块。
- `users` app 提供了后端用户 CRUD，但前端只暴露了登录、注册、个人资料。
- 真正的管理入口仍然部分依赖 Django Admin：`/admin/`。

### 5.3 “谁是配置中心”也不是单一答案

- AI 模型、提示词、生成配置：归 `requirement_analysis`
- UI 环境、AI 模式：归 `ui_automation`
- APP 环境：归 `app_automation`
- Dify：归 `assistant`
- Webhook 统一通知配置：归 `core`
- API Testing 自己还有一套 `AIServiceConfig`，并没有纳入 `/configuration`

### 5.4 任务入口分散在至少五处

- `requirement_analysis`：分析任务、任务详情
- `executions`：测试计划、测试执行、执行历史
- `api_testing`：定时任务、执行日志、测试执行
- `ui_automation`：定时任务、执行记录、AI 执行记录
- `app_automation`：定时任务、执行记录、Celery + WebSocket 进度
- `core`：`run_all_scheduled_tasks` 统一调度命令当前只覆盖 API Testing + UI Automation，不覆盖 APP Automation

### 5.5 报告入口分散在至少四处

- `/ai-generation/reports`
- `/api-testing/reports`
- `/ui-automation/reports`
- `/app-automation/reports`
- 另外 AI 智能模式的 `AIExecutionReport.vue` 没有独立一级路由，而是内嵌到执行记录页

### 5.6 通知能力是“统一配置 + 分散日志”

- 配置统一收敛到 `core/notification-configs`
- 但日志分散在：
  - API Testing 通知日志
  - UI Automation 通知日志
  - APP Automation 通知日志
- 没有统一通知日志中心

### 5.7 “定时任务配置”命名与实际内容不一致

- `/configuration/scheduled-task` 页面名像“调度器配置”
- 实际页面内容是 Webhook / 飞书 / 企微 / 钉钉机器人配置
- 实际后端 app 是 `core`，不是调度器，也不是 UI Automation 专属

### 5.8 `AI 助手` 的路由归属有歧义

- 路由是 `/ai-generation/assistant`
- 但它不在 `/ai-generation` 的 `children` 中
- 功能归属也不属于需求分析，而是独立 `assistant` app

### 5.9 存在前端与后端 API 命名漂移

- `frontend/src/views/testsuites/TestSuiteList.vue` 已接入路由，但后端 `apps/testsuites/urls.py` 为空，当前只是占位页
- `frontend/src/api/ui_automation.js` 中存在若干并未在 `apps/ui_automation/urls.py` 注册的旧接口命名，如 `notification-configs`、`task-notification-settings`、`script-steps`、`page-object-elements`、`test-environments`
- 结论：当前目录里同时存在“正式能力”和“历史遗留接口封装”

# 更新日志 (Changelog)

所有该项目的显著更改都将记录在此文件中。

## [1.0.8] - 2026-04-02

### ✨ 测试设计列表统一状态与分页稳定性补强
- 测试用例列表与版本列表接入统一页面状态组件，补齐加载、空态、搜索空态、请求失败与无权限展示。
- 在列表请求后补齐“页码越界回退”兜底，避免删除或筛选后分页器落到空页不可用。
- 保持原有筛选、排序、分页、批量删除与双击编辑行为不变。
- 扩展到执行计划列表页和接口通知日志页，统一接入页面状态组件、分页越界回退以及底部分页器布局。
- 第三批主列表页继续推进到 UI 自动化通知日志页、App 自动化通知日志页、UI 自动化执行列表页和 App 自动化执行列表页，统一接入页面状态组件、表格骨架与分页越界回退。
- 继续补齐 UI 自动化报告列表页、API 请求历史页和 API 定时任务页的统一状态、统一表格骨架与分页稳定性；其中 API 请求历史表格组件也切换为复用 `UnifiedListTable`。
- 补齐 App 自动化报告列表页的套件报告与用例报告双列表状态收口，统一接入页面状态组件、分页越界回退与稳定分页器布局。
- 继续推进 App 自动化剩余主列表页，完成设备列表页、应用包管理页、元素列表页、APP 测试用例列表页和测试套件列表页的统一状态接入、统一表格骨架改造与分页稳定性补强。
- 继续推进 API 测试主列表页，完成项目管理页、环境管理页和通知管理页的统一状态接入、统一表格骨架改造与分页稳定性补强；其中环境管理页按全局环境与局部环境两套分页独立收口。
- 首页工作台正式收口到统一 dashboard 壳思路，去掉假待办、假风险和阶段性占位数据，改为以最近访问、收藏入口和核心模块入口作为首页真实能力主轴。
- 接口自动化模块继续向平台统一壳收口，项目管理页、请求历史页、环境管理页和定时任务页改为接入统一页面壳结构，并删除历史遗留的模块内顶栏容器文件。
- 按导航冻结方案收紧接口自动化模块侧边栏，只保留正式页面入口；测试报告、定时任务、通知日志和 API AI 服务配置不再作为当前模块正式导航项，但旧路由仍可直接访问。

## [1.0.5] - 2026-03-29

### ✨ 测试设计模块前端 UI 现代化与批量删除补全
- **后端 API 标准化**：补齐项目管理 (`/projects/batch-delete/`) 和版本管理 (`/versions/batch-delete/`) 的批量删除接口，统一级联删除二次确认 (`confirm`) 的校验规范。
- **前端表格 UI 升级**：统一 `UnifiedListTable` 的视觉风格，包括全量边框、浅色表头背景，以及操作列按钮改为文字链接 (`link`) 形式。
- **交互逻辑闭环**：为项目、版本、用例列表补齐第一列序号列，全量支持多选框批量操作，并统一将行双击动作设为“触发编辑”。
- **版本管理增强**：在版本列表中新增“创建人”和“用例数”字段展示。

## [1.0.4] - 2026-03-26

### 🛠 AI 生成结果批量采纳体验与性能修复
- 任务详情页的“一键采纳”新增提交中状态，点击后会立即显示处理中提示，并临时禁用批量按钮、选择框和相关结果操作，避免用户误判未生效后重复点击。
- 补充任务详情页中的英文文案，统一“采纳中 / 已提交请求 / 自动刷新”提示语义。
- 批量采纳后端新增按 `task_id` 维度的 AI 用例查重缓存，避免每条结果重复全量扫描项目用例。
- 批量采纳链路改为在事务中批处理，并把结果状态快照收敛为循环结束后统一保存，减少循环内重复 `save()` 和默认版本重复解析带来的耗时。

## [1.0.3] - 2026-03-26

### 🔆 AI 生成链路状态收口
- 新增自动 AI 评审记录对象 `TaskAutoReviewRecord`，并补齐统一 AI 自动评审入口页与列表接口。
- 需求分析页新增按项目维度的任务恢复上下文，刷新或切页后可恢复当前生成任务入口与状态感知。
- 生成任务链路引入统一前端 tracker，收口 `SSE / polling / 本地状态`，避免旧页面实例继续收流。
- 取消生成升级为应用层协作式真取消，关键阶段与最终结果写库前都会再次检查取消状态，避免 `cancelled` 被后续写回污染。
- 生成任务详情页开放全状态查看，并按任务状态与结果处理状态双重门禁控制取消、采纳、弃用等操作能力。
- 修复 AI 需求分析 / 编写测试用例在流式阶段因 async 上下文触发同步 ORM 检查而自动失败的问题，并补齐需求分析页失败态错误提示，避免页面静默失败。
